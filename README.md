# Marmoset Brain Atlas Viewer — ユーザーマニュアル

コモンマーモセット／マウス脳の空間メタボロミクス（MSI）と組織染色（HE・IF）を ROI 単位で重ね合わせて解析できる Web アプリです。ブラウザだけで動作し、インストールは不要です。

---

## 1. 起動方法

### 1-1. 公開 URL から開く
| 用途 | URL |
| --- | --- |
| 本番（main ブランチ） | https://ryoma67744.github.io/marmoset_mouse_atlas/ |
| 開発ブランチ直読みプレビュー | https://raw.githack.com/Ryoma67744/marmoset_mouse_atlas/claude/investigate-image-rotation-Ny7DQ/viewer/index.html?atlas=cor_slide_1_10 |

ルート URL を開くと自動的に `viewer/index.html?atlas=cor_slide_1_10` にリダイレクトされ、サンプルデータセットが読み込まれます。

### 1-2. ローカルで動かす（オフライン確認）
```
python3 -m http.server 8000
```
を `marmoset_mouse_atlas/` 直下で実行し、`http://localhost:8000/` にアクセスします。

### 1-3. データセットを切り替える
クエリパラメータ `?atlas=<フォルダ名>` で切り替え可能:
- `?atlas=cor_slide_1_10` … コロナルスライス 1/10（HE TIFF + MSI 4 化合物）
- `?atlas=test1` … HE/IF/Mam-Sagittal を含むサンプル

### 1-4. 最新版の確認
画面右下に **v:xxxxx** の小さなビルドタグが表示されます（例: `v:rot-fix-2`）。ブラウザキャッシュで古い版が出ているときは、`Ctrl+F5` (Windows) または `Cmd+Shift+R` (macOS) で強制再読み込みしてください。

## 3. Image Viewer の使い方

### 3-1. 画像を選ぶ
右上の **Select Images** ボタンをクリック → チェックボックスで表示したい画像を選択。ボタン表記はチェックした画像名（例: `HE Stain + MSI (DA) + MSI (NE)`）に切り替わります。

サムネイル（IMAGES 領域）を **クリック** しても同じ選択トグルが働きます。選択中のサムネには青枠が付きます。

### 3-2. 重ね合わせの仕様
- **HE / IF** は通常のグレースケール正規化で描画されます。
- **MSI** は加算合成 (`lighter` ブレンド) で描画され、複数の MSI を同時選択すると色が混ざります。
- 各化合物はあらかじめ決まった色で着色されます: DA=赤、5-HT=緑、NE=黄、Ach=橙、GABA=シアン、Glu=マゼンタ、TIC=無彩色。

### 3-3. ROI オーバーレイ
ROI LIST で **Show** トグルが ON のとき、選択可能な ROI 輪郭が Image Viewer 上に重なります。`+新規` で手描き追加した ROI も同様に表示されます。

### 3-4. ROI の手描き登録
1. ROI LIST 上部の **`+新規`** をクリック。ボタンが `中止` に変わり、Image Viewer のカーソルが十字に変わります。
2. Image Viewer 内を順にクリックして頂点を追加していきます（3 点以上）。
3. **最初の頂点をもう一度クリック** するか、**ダブルクリック** / **Enter キー** で確定。名前入力ダイアログが出ます。
4. **Escape** キーで中止。
5. 作成した ROI は `User ROI 1, 2, …` として ROI LIST に登録され、ダウンロード時の xlsx / json に列・フィールドとして出力されます。

---

## 4. ROI LIST の使い方

| 操作 | 目的 |
| --- | --- |
| ROI 名行をクリック | その ROI をハイライト、ANALYSIS グラフを更新 |
| 左のチェックボックス | この ROI を表示/非表示 |
| `Show` トグル | 全 ROI の一斉表示/非表示 |
| `+新規` | 手描き ROI 追加モード（上記 3-4） |

色と名前はデータセット提供側で定義され、手描き ROI は衝突しないパレットから自動的に色が割り当てられます。

---

## 5. IMAGES（サムネイル）の使い方

- 実画像があるソースのみ **横 1 列** で並びます（`No Image` プレースホルダは非表示）。
- サムネイル右下部のラベル行には化合物名とカラードットが付きます。
- **クリック** で Image Viewer への追加/解除をトグルできます。
- **ホバー** すると歯車メニューが出て（MSI のみ）、`vmin` / `vmax` スライダで個別にコントラスト調整ができます。調整結果は Image Viewer とサムネの双方に即時反映されます。
- サムネは Image Viewer と同じ向き（解剖学的に一致した向き）で表示されます。

---

## 6. ANALYSIS グラフの使い方

1. 上部の 3 つのプルダウンで比較する化合物を選択します（`-` を選ぶとその系列は表示されません）。
2. ROI LIST から ROI をクリックするとグラフが更新されます。
3. 各バーはその ROI 内の平均強度（Mean Intensity）です。

グラフは Image Viewer で選択中の画像と連動しており、色は化合物に対応した MSI カラーです。

---

## 7. Download（ZIP バンドル出力）

ヘッダー右の **Download** ボタンで以下を含む ZIP を生成します:

- `atlas.json` … ROI 定義・world 座標・画像参照を含むメタファイル
- `data/*.xlsx` … 元 MSI データに **手描き ROI 列を追加**した版（差し替え失敗時はオリジナルのまま）
- `data/*.TIF` / `data/*.json` … ソースの付随ファイル（存在する場合）

研究データとして **手描き ROI も含めてそのまま配布/保存** できます。ROI 座標は MSI ピクセル空間（生の列・行インデックス）で記録されるため、画面表示の回転や拡大縮小には依存しません。

## 8. データを自前で追加する

各データセットは `datasets/<atlas_id>/` 直下に配置し、以下を含めます:

```
datasets/<atlas_id>/
├── atlas.json        # 必須。ROI / 画像参照 / world 座標
└── data/
    ├── HE_Mam_...TIF # HE 画像 (TIFF も PNG も可)
    ├── *.xlsx        # MSI 生データ (sheet, 列指定を atlas.json に記述)
    └── *.json        # 任意のメタ
```

`atlas.json` のキー:
- `meta` – 検体情報（ID, Organism, Strain, Age, Sex）
- `roi.roi_items` – ROI ポリゴン座標（MSI ピクセル空間）
- `roi.palette` – ROI キー → RGBA 色
- `roi.roi_names` – ROI キー → 表示名
- `images` – 画像キー → 相対パス / data URI
- `xlsx_series` – MSI 生データ → PNG 化ルール（`src` / `sheet` / `col_x` / `col_y` / `col_v` / `range`）
- `thumbnail_order` – IMAGES の並び順指定（配列）
- `world_coords` – `he_um_per_px`, `msi_um_per_px`, `T_he_to_msi` 等

追加後は `?atlas=<atlas_id>` でアクセスします。

## 10. トラブルシューティング

| 症状 | 対処 |
| --- | --- |
| **最新の修正が反映されない** | 右下ビルドタグを確認。古い値なら `Ctrl+F5` / `Cmd+Shift+R`。それでも古ければ Pages デプロイ完走を 1–2 分待つ |
| **TIFF が表示されない** | ブラウザが UTIF.js の読み込みに失敗している可能性。オフライン環境では CDN 到達性を確認 |
| **MSI 画像が真っ黒** | `vmin` / `vmax` がレンジ外。サムネイルのホバー歯車で調整 |
| **ROI を描いてもクリック位置がずれる** | 画面ズーム 100% にして再試行。CSS 変形を通じてマウス座標はローカル系で再計算されるため 100% で最も正確 |
| **ZIP ダウンロードに時間がかかる** | 大きな TIFF / xlsx を同梱するため。進捗表示は出ませんが、ブラウザの通信インジケータで進行中か判断 |
| **`Select Images` で何もチェックできない** | その atlas に画像が一つも登録されていません。`atlas.json` の `images` / `xlsx_series` を確認 |

## 11. 既知の仕様・制約

- Image Viewer は CSS で −90° 回転、内部キャンバスでさらに −90° 回転しており、ソース画像は **ソースから −180°** の向きで表示されます（解剖学的には適切な向き）。
- HE/IF は `T_he_to_msi`（≈180° 回転含むアフィン）で MSI 座標系に合わせてから描画されるため、サムネと Image Viewer で一貫した向きになります。
- `Mass 測定情報` エリアは今後の機能拡張用で、現状は案内表示のみ。
- IE11 等のレガシーブラウザは非対応。Chrome / Edge / Firefox / Safari 最新版を推奨。
- モバイル（狭幅）表示は未最適化。1024px 以上の横幅で利用してください。
